# 上报样本辅助脱敏设计

## 背景

当前“上报脱敏样本”页面要求用户手动编辑短信正文，自行删除或替换姓名、手机号、地址、身份证号、订单号、银行卡号等敏感信息。这个流程成本高，尤其是从会话详情带入真实短信后，用户需要逐字检查并手动构造替换内容，容易放弃提交，也容易遗漏敏感数据。

第一版目标是降低用户脱敏成本，但不改变样本上报的隐私边界：应用只提供本地辅助替换工具，不自动上传原文，不引入远程解析，不替用户做最终隐私承诺。

## 目标

- 在 `SampleSubmissionScreen` 中提供 BigBang 风格的辅助脱敏入口。
- 用户可以选择短信中的一个词语或连续文本片段，并按敏感类型替换成虚拟假数据。
- 假数据保留原始类型和大致格式，让样本仍然适合后续规则生成和模型改进。
- 支持连续多次替换，直到用户确认将脱敏后的工作副本应用回输入框。
- 全流程在端侧完成，不新增网络依赖，不改变现有提交接口。

## 非目标

- 不在第一版引入中文 NLP、云端识别或外部实体抽取服务。
- 不承诺自动识别所有敏感数据。
- 不自动勾选“我确认已脱敏”复选框。
- 不改变 PixelText Hub 的样本提交协议。
- 不在本次设计中实现服务端校验或样本质量评分。

## 现有上下文

现有实现包含：

- `SampleSubmissionScreen.kt`：负责发件人、样本类别、短信样本输入、隐私确认和提交按钮。
- `SampleSubmissionViewModel.kt`：维护 `content`、`sender`、`category`、`agreed`、`submitting` 和提交结果。
- `SampleSubmissionRepository.kt`：提交用户确认后的样本文本、发件人、类别、版本信息和 `ANDROID_ID`。
- `AppNavigation.kt`：支持从设置页进入样本上报页，也支持从会话详情带入单条短信正文和发件人。

本设计只扩展样本上报页和本地脱敏逻辑，不改动上传仓库和 Hub client。

## 推荐方案

采用“半自动辅助脱敏”：

1. 页面提供 `辅助脱敏` 按钮。
2. 用户点击后打开底部面板。
3. 面板展示当前短信内容的 BigBang 视图。
4. 用户选择一个或连续多个文本片段。
5. 用户选择敏感类型。
6. 点击 `替换`，系统生成 fake 数据并更新面板中的工作副本。
7. 用户可以继续替换其他片段。
8. 点击 `应用到样本` 后，工作副本回写到原短信输入框。

该方案不引入重量级分词依赖，交互成本低，和当前 Compose 页面结构匹配。强格式数据可以在后续实现中用本地正则做自动高亮，但第一版仍以用户选择为核心。

## 架构

新增本地领域能力，避免把脱敏规则写进 UI：

```text
domain/sample/
├── SampleTextTokenizer.kt
├── FakeSampleGenerator.kt
└── SampleDesensitizer.kt
```

### SampleTextTokenizer

职责：

- 将短信内容切成可选择片段。
- 为每个片段记录原文 `start` 和 `end` 偏移量。
- 保留标点、空白和换行的展示信息。
- 支持中文连续片段按单字进一步选择，方便选中人名和地址中的局部内容。

第一版切分策略：

- 连续数字作为一个片段。
- 连续字母数字作为一个片段。
- 连续中文作为一个片段，但 UI 可按单字选择子范围。
- 标点、空格、换行作为分隔或不可替换片段展示。

### FakeSampleGenerator

职责：

- 根据敏感类型和原始文本生成虚拟假数据。
- 尽量保留原始长度、字符类别和业务可读性。
- 不使用真实用户数据、不读取通讯录、不读取短信库额外内容。

支持类型：

- `姓名`
- `手机号`
- `身份证号`
- `地址`
- `银行卡号`
- `订单号`
- `验证码`
- `金额`
- `其他`

### SampleDesensitizer

职责：

- 接收原文、选区和敏感类型。
- 调用 `FakeSampleGenerator` 生成替换文本。
- 返回替换后的新文本。
- 保证非选中内容保持不变。

## ViewModel 状态

`SampleSubmissionViewModel` 继续维护现有提交状态，并新增辅助脱敏相关状态：

- 是否显示辅助脱敏面板。
- 面板工作副本内容。
- 当前分片列表。
- 当前选区。
- 当前敏感类型。
- 是否存在未应用修改。

建议提供的方法：

- `openDesensitizationAssistant()`
- `closeDesensitizationAssistant()`
- `selectSensitiveRange(start: Int, end: Int)`
- `updateSensitiveType(type: SensitiveType)`
- `replaceSelectedSensitiveText()`
- `applyDesensitizedDraft()`
- `discardDesensitizedDraft()`

`applyDesensitizedDraft()` 回写 `content` 后应清空 `agreed`，让用户重新确认最终内容。

## UI 设计

`SampleSubmissionScreen` 中在短信输入框附近增加 `辅助脱敏` 按钮。当前内容为空时按钮禁用。

点击后打开 `ModalBottomSheet`：

- 顶部展示标题 `辅助脱敏`。
- 主体展示 BigBang 分片视图。
- 底部展示敏感类型选择。
- 有选区和类型后启用 `替换` 按钮。
- 展示替换后的预览文本。
- 提供 `应用到样本` 和 `取消` 操作。

选择规则：

- 第一版只支持连续范围选择。
- 点击片段开始选择，再点击后续片段扩展选择。
- 对中文长片段，UI 允许按单字选择连续子范围。
- 替换后重新分片，并清空当前选区。

关闭规则：

- 没有未应用修改时直接关闭。
- 有未应用修改时弹出确认：`放弃修改` / `继续编辑`。
- `应用到样本` 后关闭面板并回写输入框。

## Fake 数据规则

生成规则以“格式相似、内容虚拟”为原则：

- 姓名：生成常见中文假名，长度尽量接近原文。
- 手机号：生成 `13`、`15`、`17`、`18`、`19` 开头的 11 位号码。
- 身份证号：生成 18 位数字格式，第一版保证格式正确，严格校验位可后续补充。
- 地址：生成虚拟省市区、路名、小区、楼栋和门牌组合。
- 银行卡号：生成 16 位或 19 位数字，长度尽量贴近原文。
- 订单号：按原文字符类型生成同长度数字或字母数字串。
- 验证码：按原文字符类型生成同长度数字或字母数字串。
- 金额：保留金额符号、单位和小数格式，例如 `123.45元` 替换为 `86.20元`。
- 其他：按原字符类型生成同长度占位文本。

假数据生成应使用本地随机源，不写入日志，不上传映射关系。

## 错误处理

- 内容为空时禁用 `辅助脱敏`。
- 没有选中文本时禁用 `替换`。
- 没有选择敏感类型时禁用 `替换`。
- 替换后选区失效时清空选区并刷新分片。
- 工作副本为空时不允许应用到样本。
- 关闭有未应用修改的面板时要求确认。

## 隐私说明

辅助脱敏只处理当前输入框中的文本。打开面板、选择片段、生成 fake 数据和回写样本都在本地完成。

该功能不会：

- 自动上传原始短信。
- 将原文发送给服务端识别。
- 读取通讯录或其他短信来生成假数据。
- 替用户完成最终隐私确认。

提交按钮仍然依赖用户勾选确认文案。该确认文案需要继续说明用户应检查并确认姓名、手机号、地址、订单号、银行卡号等敏感信息已经删除或替换。

## 验证计划

单元测试覆盖：

- tokenizer 对数字、字母数字、中文、标点、空白和换行的切分。
- 姓名、手机号、身份证号、地址、银行卡号、订单号、验证码、金额和其他类型的 fake 数据生成。
- `SampleDesensitizer` 替换单个范围后非选中内容不变。
- 连续多次替换后重新分片，偏移量正确。
- 空内容、空选区、非法选区不会产生错误替换。

编译检查：

```bash
./gradlew :app:compileDebugKotlin
```

手动验证：

- 从设置页进入上报样本页，输入短信后打开辅助脱敏。
- 从会话详情选择单条短信上报，确认原文带入后可以辅助脱敏。
- 替换姓名、手机号、地址、订单号等片段后应用到样本。
- 取消未应用修改时确认输入框内容不变。
- 应用脱敏结果后确认隐私复选框未被自动勾选。

## 实施顺序建议

1. 新增 `domain/sample` 下的 tokenizer、fake generator 和 desensitizer。
2. 为本地脱敏逻辑补单元测试。
3. 扩展 `SampleSubmissionViewModel` 的辅助脱敏状态和操作。
4. 在 `SampleSubmissionScreen` 增加按钮、底部面板、分片选择和替换预览。
5. 跑 `./gradlew :app:compileDebugKotlin`，并进行页面手动验证。
